## Назначение приложения

**SHAT Docs** - это система для управления учебными группами, студентами, посещаемостью и успеваемостью с веб-интерфейсом и API.

## Основные разделы

### Главная страница (`/`)

- карточки быстрых переходов в личный кабинет, группы и документацию;
- блок с последними релизами (мажорные, бета, патч). При отсутствии интернета отображается сетевой fallback;
- диалоги с деталями изменений по релизам.

### Личный кабинет (`/profile`)

- просмотр профиля пользователя;
- статистика по студентам, посещаемости и среднему баллу;
- изменение ФИО, email и пароля;
- просмотр активных сессий (устройств) и завершение отдельных сессий;
- отправка заявки на сброс пароля в админ-панель.

### Группы (`/profile/groups`)

- список **ваших** групп (в v2 чужие группы API не отдаёт);
- создание новой группы (**одна группа на преподавателя**);
- переход в карточку конкретной группы.

### Управление группой (`/profile/groups/[id]`)

- редактирование названия группы (только владелец);
- передача прав владения другому преподавателю (список преподавателей подгружается при открытии диалога);
- удаление группы;
- вкладки с **ленивой загрузкой** (dynamic import + данные при первом открытии):
  - **Посещаемость**
  - **Успеваемость**
  - **Студенты** (список студентов запрашивается при первом переходе на вкладку)

### Поддержка (`/support`)

- единый центр помощи для пользователей;
- каналы связи: email, Telegram, GitHub Issues;
- рекомендации, какие данные приложить к обращению для быстрого решения проблемы.

## Типовой рабочий сценарий преподавателя

1. Авторизоваться через `/login`.
2. Перейти в `/profile/groups` и открыть нужную группу.
3. Заполнить или загрузить посещаемость.
4. Заполнить оценки по студентам.
5. Проверить список студентов и внести изменения при необходимости.
6. При подозрительной активности завершить лишние сессии в `/profile`.

## Доступ и безопасность (v2)

- `/profile/*` и `/admin/*` защищены **middleware** (проверка JWT в cookie до отдачи страницы);
- API проверяет **владельца группы** (или `canAccessAdmin`) на attendance, grades, students, stats;
- при отсутствии сессии — редирект на `/login?to=...`;
- API поддерживает cookie-сессию и `Authorization: Bearer <token>`;
- мутации требуют **CSRF-токен** (`GET /api/auth/csrf` → заголовок `x-csrf-token`);
- rate limit на login, register и большинство API-маршрутов;
- смена пароля завершает другие активные сессии пользователя (кроме текущей);
- вкладки профиля и группы поддерживают **ARIA** и навигацию с клавиатуры (стрелки, Home/End).

## Регистрация

Регистрация работает как заявка:

- после `/register` аккаунт получает статус `pending`;
- войти можно только после подтверждения администратором;
- админ управляет заявками в `/admin`.

## Сброс пароля через заявку

Пользователь может отправить заявку на сброс пароля из личного кабинета (`/profile`).

- создается заявка в статусе `pending`;
- повторная активная заявка не допускается;
- администратор в `/admin` может:
  - подтвердить сброс (`resolved`) и задать новый пароль;
  - отменить заявку (`cancelled`).

Подробнее: [Регистрация и статусы аккаунта](/wiki/registration).
