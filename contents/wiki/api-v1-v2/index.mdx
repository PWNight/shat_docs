## Главные отличия

### 1) Единая инфраструктура ответов в v2

В `v2` введён набор утилит:

- `successResponse(...)`, `errorResponse(...)`;
- `badRequest(...)`, `notFound(...)`, `unauthorized(...)`, `serverError(...)`;
- `handleApiError(...)`.

Это даёт предсказуемые ответы и единый формат ошибок во всех роутингах.

В `v1` обработка ошибок реализована локально в каждом route, из-за чего структура ответа может немного отличаться между эндпоинтами.

### 2) Безопасный парсинг JSON и валидация

В `v2` добавлены:

- `safeParseJson(request)` - защита от невалидного JSON;
- `validateData(schema, data)` - унифицированная валидация через Zod.

В `v1` JSON и валидация обычно обрабатываются напрямую в каждом обработчике.

### 3) Авторизация: cookie + Bearer в v2

`v2` поддерживает два источника авторизации:

- сессия из cookie;
- `Authorization: Bearer <token>`.

Это реализовано через `extractSession(...)` и особенно полезно для внешних клиентов (боты, сервисы, интеграционные скрипты).

В `v1` маршруты ориентированы в основном на cookie-сессию.

### 4) Изменения в auth/register

В `v2` после регистрации возвращается также токен сессии:

- `data: { uid, email, full_name, token }`

В `v1` токен в ответе регистрации не возвращался.
