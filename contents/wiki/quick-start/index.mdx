## 1. Регистрация и вход

1. Откройте страницу `/register` и создайте аккаунт.
2. После регистрации заявка получает статус `pending` и требует подтверждения администратором.
3. После подтверждения вход выполняется через `/login`.

Подробнее: [Регистрация и статусы аккаунта](/wiki/registration).

## 2. Проверка профиля

На странице `/profile` доступны:

- краткая статистика по ученикам и посещаемости;
- редактирование ФИО и email;
- смена пароля;
- список активных сессий (устройств) с возможностью завершать их по одной;
- отправка заявки на сброс пароля (для обработки администратором).

Перед началом работы проверьте актуальность ФИО и email, так как они используются в интерфейсе и API-операциях.

## 3. Создание первой группы

1. Перейдите в `/profile/groups`.
2. Нажмите **Создать группу**.
3. Укажите название группы и сохраните.

**v2:** у одного преподавателя может быть **только одна** закреплённая группа. Если группа уже есть, создание второй вернёт ошибку.

После создания карточка группы появится в списке **ваших** групп (чужие группы в списке не отображаются).

## 4. Работа внутри группы

Откройте нужную группу по кнопке **Подробнее**. Доступны три вкладки (в v2 код и данные подгружаются **при первом открытии** вкладки):

- **Посещаемость** — импорт/редактирование/сохранение данных;
- **Успеваемость** — оценки по предметам и средний балл;
- **Студенты** — список, добавление, изменение и удаление.

## 5. API и техническая документация

- API Reference (v3): `/wiki/api-reference` (или `/api`)
- Сравнение API: [v2 vs v3](/wiki/api-v2-v3)

## Для разработчиков (v2)

```bash
npm run dev          # локальный сервер
npm run typecheck    # проверка TypeScript
npm run test         # Vitest
npm run backup:db    # резервная копия SQLite
curl /api/health     # статус приложения и БД
```

## Частые проблемы

### Не пускает на защищенную страницу

В v2 маршруты `/profile/*` и `/admin/*` защищены **middleware** на сервере. Если сессия истекла или недействительна, произойдёт редирект на `/login` с параметром `to` для возврата.

### Ошибка сохранения данных

Проверьте обязательные поля и формат входных данных. В новых версиях приложения сообщения ошибок унифицированы и показываются в уведомлениях.

### Нужен сброс пароля

Из профиля отправьте заявку на сброс пароля. Заявка попадет в админ-панель и будет обработана администратором (подтверждена или отменена).
