## Кому это нужно

Эта страница предназначена для пользователей с доступом к админ-панели (`/admin`).

## Доступ в админ-панель

Админ-панель доступна только пользователям, которые:

- авторизованы (есть сессия), и
- имеют право доступа `canAccessAdmin` (выдаётся администратором).

Если у пользователя нет доступа, API вернёт `403 FORBIDDEN`.

## Структура админ-панели

В админ-панели есть несколько ключевых зон:

- **Обзор (статистика)**: общее количество пользователей, групп, студентов, ожидающих регистраций и заявок на сброс пароля.
- **Группы**: создание, редактирование, удаление групп, назначение преподавателя (владельца группы).
- **Пользователи и безопасность**:
  - заявки на регистрацию (`pending`) → подтвердить/отклонить,
  - управление доступом к админ-панели,
  - прямой сброс пароля,
  - удаление пользователей (с ограничениями).
- **Логи**: аудит действий администраторов (что, кто и когда сделал).

## Регистрация пользователей (заявки)

Новые регистрации создаются со статусом `pending`.

Действия администратора:

- **Подтвердить**: пользователь становится `approved` и сможет войти.
- **Отклонить**: пользователь становится `rejected` и не сможет войти (также сбрасываются админ-флаги, если были выставлены).

Технически это выполняется через API:

- `POST /api/admin/registrations/{id}/approve`
- `POST /api/admin/registrations/{id}/reject`

## Управление доступом к админке

Для выдачи/отзыва доступа используется:

- `POST /api/admin/users/{id}/access`

Ограничения:

- нельзя менять доступ самому себе;
- нельзя менять права root-пользователя.

## Управление пользователями

Поддерживаются операции:

- обновление пользователя (ФИО и email);
- удаление пользователя;
- прямой сброс пароля.

Ограничения:

- нельзя удалить самого себя;
- нельзя удалить root-пользователя.

## Управление группами

Админ может:

- создать группу (название + преподаватель);
- изменить название и/или преподавателя;
- удалить группу.

Удаление группы удаляет связанные данные (посещаемость/успеваемость/студентов), поэтому используйте аккуратно.

## Ошибки: обычная vs недоступность БД

Админ-панель и API различают:

- **обычную ошибку запроса** (`500` и другие статусы),
- **недоступность БД**: `HTTP 503` + `code: "DB_OFFLINE"`.

Если БД недоступна, UI покажет отдельное состояние и предложит повторить попытку после восстановления соединения.
